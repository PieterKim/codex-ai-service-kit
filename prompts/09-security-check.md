# 09. 보안 점검 프롬프트

아래 내용을 채운 뒤 Codex에 그대로 붙여넣어 사용하세요.

```text
GitHub에 프로젝트를 올리기 전에 민감정보와 보안 위험을 점검해줘.

## 점검할 프로젝트
- 프로젝트 경로:
- 사용한 AI API:
- 사용한 외부 서비스:
- 개인정보 또는 사용자 업로드 데이터 사용 여부:

## 특히 확인할 항목
- API Key
- token
- password
- credentials.json
- auth.json
- token.json
- .env
- 로그 파일
- 업로드 파일
- 개인정보 포함 데이터

## 요청 사항
다음 기준으로 점검해줘.

1. GitHub에 올리면 안 되는 파일이 있는지 확인
2. `.env`와 `.env.example`이 올바르게 구분되어 있는지 확인
3. 코드에 API Key, token, password가 직접 작성되어 있는지 확인
4. README와 보고서에 실제 비밀값이 있는지 확인
5. 개인정보 포함 데이터가 저장소에 들어가 있는지 확인
6. `.gitignore`에 필요한 항목이 포함되어 있는지 확인
7. GitHub push 전 최종 체크리스트를 작성

민감정보가 발견되면 실제 값을 출력하지 말고, 파일 위치와 조치 방법만 알려줘.
```
