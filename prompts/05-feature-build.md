## 구현 기준
- 기존 프로젝트 구조를 먼저 확인해줘.
- 관련 파일을 읽고 필요한 파일만 수정해줘.
- 한 번에 전체 시스템을 만들지 말고 이 기능에 필요한 최소 범위만 구현해줘.
- router, schema, service 역할을 가능하면 분리해줘.
- 실제 API Key, token, password는 코드에 직접 작성하지 말아줘.

## 요청 사항
구현 후 다음 내용을 알려줘.

- 수정한 파일
- 구현한 내용
- 실행 방법
- 테스트 또는 수동 확인 방법
- 다음에 구현하면 좋은 기능
